rsi-maker-loop

Self-correcting loop — decompose, then correct errors at every step (MAKER, arXiv:2511.09030).
voters: cheap (Haiku) judge: strong vote: first-to-ahead-by-k (k≈3) red-flag: malformed / over-long
01 · INTAKE
Pick the work
User-reported bugs first, else the first unchecked BACKLOG.md item (P0→P3).
↓
02 · DECOMPOSE
Break into tiny checkable micro-steps
Each step small enough to verify; classify it as verifiable or a judgment call.
↓  per micro-step  ↓
Verifiable
The test is the judge
Generate → run tests / typecheck / build → accept what passes. No ensemble (free, strongest).
Unverifiable
first-to-ahead-by-k vote
Diverse independent samples; red-flag bad ones; accept when one leads by k. Else a judge decides.
Haiku ×k+ minimality voter
↓
03 · COMPOSE & GATE
Full quality gate — commit only if green
Assemble accepted steps, run the complete suite + typecheck + build. Red → fix or revert.
↓
04 · COMMIT & REGENERATE
Record, commit, refill the backlog
Update BACKLOG.md + docs; commit per target. When empty, propose 3–5 items (human checkpoint).
↺
↻ schedule next iteration · perpetual
rsi-maker-loop execution errors bounded, not strategic drift — direction stays human
