rsi-loop

Lean autonomous backlog loop — one agent, the repo's test gate as the safety net.
1 agent / iteration cost: low model: one strong (Opus / Sonnet) gate: tests · typecheck · build
01 · INTAKE
Pick the work
User-reported bugs first, else the first unchecked BACKLOG.md item (P0→P3).
↓
02 · IMPLEMENT
Minimal correct change
One capable model, reuse existing modules, no speculative abstractions. Stay within blast radius.
↓
03 · GATE
Full quality gate — commit only if green
Run the complete suite + typecheck + build. Red → fix or revert, never commit red.
↓
04 · RECORD & COMMIT
Check off, commit per target
Update BACKLOG.md + docs. Commit to branch + PR (default) or main.
↓
05 · REGENERATE (RSI)
Backlog empty? make 3–5 new items
High-value, non-speculative. Surface to the human before going deep — direction stays theirs.
↺
↻ schedule next iteration · perpetual
